01 Interaction model

Trigger: the card fires only when a selection settles — mouse released (or keyboard selection idle) and unchanged for 300 ms, span non-empty after trimming. Dragging never fires; every selection change during flight cancels and supersedes (generation token, mirroring phoneticGeneration). Re-selecting the identical span is a no-op — the existing card stays.

Fig. 1 — Sequence: settle-then-fire, one in-flight request, generation token discards stale responses (PRD AC-7).

Coexistence with the panel's existing affordances

Fig. 2 — Click map. The card becomes the third "safe" zone in the existing outside-click collapse; everything else behaves exactly as v1.2.10.
  • ⌘S routing: while a card is active, ⌘S saves the card (span → source, contextual translation → translation, FR-7); the header button visually stays but the key equivalent moves to the card's "☆ Save ⌘S" control, and moves back on dismissal. One shortcut, context-dependent target — matching what the user is looking at.
  • Retranslate / new capture / error / close: all funnel through the existing teardown paths (dropResultBody, setRetranslating) — each cancels the in-flight span request and clears the slot before content changes, so a stale card can never sit over new content (FR-6).
  • Selection in the Translation pane: not a trigger (out of scope) and dismisses an active card — one active lookup at a time.

02 Card design

Anchoring — fixed slot docked below the Recognized pane; the panel grows downward (top edge fixed, growth ≤ 200 pt, clamped to the screen's visible frame). A selection-anchored popover inside a 440-pt panel would routinely cover the very words being examined or clip at the panel edge; a fixed slot never occludes anything, lands directly under the pane the user is already looking at, and gives VoiceOver a stable landmark. Growth animates 180 ms ease-out; instant under Reduce Motion.
Fig. 3 — Card anatomy and variants. One frame, one hierarchy; variants differ only in chip, border style, and which rows exist.

06 Accessibility & edge notes

  • VoiceOver: card is one accessibility group (“Contextual translation”), children in reading order: span → mode chip → translation → POS → sense → example. On render, post an announcement: “scored — 攻入, verb”. ✕ is labeled “Dismiss contextual translation”; Save mirrors the header button’s labeling. The skeleton sets aria-busy-equivalent (AX busy) so VO doesn’t read placeholder bars.
  • Keyboard: selection via ⇧←/→ works — same settle debounce. Esc collapses selection + card from any first responder in the panel. ⌘S targets the card while active (see §01).
  • Reduce Motion: no shimmer (static bars), no grow animation (instant height), no announcement change.
  • Very long words / spans: span echo is single-line, middle-truncated; card translation wraps up to 3 lines then truncates (like the IPA cap, ResultPanel.swift:814); card max height 200 pt, internal content never scrolls — long content belongs to long-span mode’s plain block, which wraps freely.
  • zh selections (no spaces): word/phrase mode by grapheme count (≤ 8) since whitespace tokens are always 1; boundaries come from the user’s selection, never a tokenizer (AC-9). POS/example may be absent for zh spans — rows omit, never fake (FR-3).
  • Multi-line selections: span is whitespace-normalized (newlines → single space) for both the request and the echo.
  • Whitespace/punctuation-only selections: trimmed empty → no request, no slot.
  • Panel near screen edge: growth is clamped to the screen’s visibleFrame; if there’s not enough room below, the panel shifts up by the shortfall before growing — the card is inside the panel, so it can never clip offscreen (the popover failure mode this anchoring avoids).
  • Google-only + long span: degraded chip + plain block merge (degraded styling wins); still no error.

07 Open questions for Gate 1

Q1 · ⌘S routing. While a card is active, ⌘S saves the card instead of the whole result (key equivalent swaps to the card and back on dismissal). Confirm this override is acceptable vs. a separate shortcut.
Q2 · CJK threshold. Word/phrase mode for unspaced scripts = ≤ 8 graphemes (tunable). Confirm, or pick a different cap.
Q3 · Card read-aloud. The card ships without its own speaker (pane speakers keep reading whole panes). A span-speaker on the card is a cheap fast-follow — in or out of v1?
