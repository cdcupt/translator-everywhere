4 · Manual E2E — the human beta (gates release)

Why manual: the core value — a key syncing across two Macs — can't be exercised headlessly: it needs the backend deployed with the master key, real Google/Apple sign-in, the login Keychain under a real session, and two machines. A headless "beta panel" would only re-run the same UI logic the 211 tests already cover. So the honest human beta is this checklist, run once after deploy:
  • AC1 round-trip: Mac A — enter sk-…, toggle "Sync this key across my Macs" on, confirm it shows Synced. Mac B (clean Keychain, same account) — sign in → key auto-restored + "Restored" confirmation; run the Engine Test → passes.
  • Don't-clobber: Mac B already has a different key → sign in → conflict prompt appears; "Keep this Mac's key" leaves it unchanged.
  • Opt-out: toggle off on A → on B (or an authed GET) the server returns 404.
  • Encrypted-at-rest (AC2): psql → SELECT encode(blob,'hex') FROM user_secrets LIMIT 1; is opaque (not sk-…); confirm SECRET_ENCRYPTION_KEY lives only in ~/.translator-everywhere/deploy.env (chmod 600), not the DB/repo.
  • No plaintext in prod logs (AC3): after a real PUT/GET, grep the container logs — the key never appears.
  • 503 degrade: (optional) with the master key unset, /secret/* returns 503 and the rest of the app still works.
