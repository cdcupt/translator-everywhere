Translator Everywhere — macOS app

Wrap the te screen-capture translator in a native menu-bar app that anyone can download from GitHub, install by dragging to Applications, and use via a global hotkey that works without touching System Settings. The whole point is a zero-friction "capture anything on screen → translate" experience for non-technical users.

1Problem & why now

The te CLI already does screen-capture translation (te -c): drag a region, OCR + translate, popup + clipboard. But it only serves a technical user on this Mac:

  • Install is developer-only — clone a repo, run install.sh, have jq and an OpenAI key. A non-technical user can't.
  • The hotkey can't be pre-set — macOS forbids a Quick Action from shipping with a keyboard shortcut, so every user must hand-bind it in System Settings → Services. That is exactly the friction we want gone.
  • It needs the author's paid OpenAI key — there's no per-user, no-cost path.

A native menu-bar app removes all three at once: a downloadable bundle, an app-registered global hotkey, and a free on-device default engine.

2Users & the core job

Primary — the non-technical Mac user

Watches videos / reads pages in a second language. Wants to translate anything on screen (a YouTube frame, an image, a PDF, a non-selectable UI) with one gesture. Will not open a terminal.

Secondary — the power user (Erik)

Wants the higher-quality OpenAI path and a configurable hotkey, but the same one-press flow. Keeps the existing te CLI too.

Core job-to-be-done: "When I see text on my screen I can't read, let me press one key, draw a box around it, and instantly get a translation I can read and paste."

3Goals & success criteria

#Success criterion (measurable)
G1A new user goes from downloaded .dmg → working translation in < 2 minutes, no terminal, no API key.
G2The global hotkey is set entirely inside the app (a Preferences field), never in System Settings.
G3Default engine produces a usable translation with zero configuration and zero cost (Vision OCR + Google).
G4Capture→result round trip feels instant: < ~3 s for the free path on a normal region.
G5First-run onboarding gets the user through the Screen Recording permission without confusion.
G6The app is downloadable from GitHub Releases and opens on a clean Mac (with at most one documented right-click→Open step while unsigned).

5The engine decision Gate-1 sign-off

⚑ Coordinator recommendation — "Both, switchable" with a FREE default

The current te -c uses OpenAI vision, which requires each user's paid key — a non-starter for free distribution. For the app I recommend:

  • Default (no setup): Apple Vision OCR (on-device, free, private) → Google Translate (free public endpoint). Works the instant the app opens, no key.
  • Optional upgrade: a Preferences field to paste an OpenAI key (stored in the macOS Keychain) for higher-quality AI translation. Off by default.

This needs your explicit approval at Gate 1, because it changes the default engine away from today's OpenAI-vision path. Alternatives on the table: OpenAI-only (best quality, worst onboarding) or free-only (simplest, no AI upgrade).

7Scope

In — v1 must

  • Menu-bar app (no Dock app) with a status item + menu.
  • "Capture & Translate" command + app-registered global hotkey.
  • Region capture → Vision OCR → translate → popup + clipboard.
  • Preferences: hotkey, engine (free / OpenAI key), translation direction default.
  • First-run onboarding incl. Screen Recording permission guidance.
  • OpenAI key stored in Keychain (never on disk in clear).
  • Packaged .dmg published to GitHub Releases; README install guide.

Should if cheap / Later v2

  • should "Launch at login" toggle.
  • should Translate-clipboard & translate-selection menu items (reuse te logic).
  • later Developer-ID signing + notarization (no Gatekeeper warning).
  • later History of recent translations; in-place overlay instead of a dialog.
  • later Auto-update.

Out not now

  • App Store distribution / sandboxing.
  • iOS / Windows / Linux.
  • Replacing the te CLI (it stays as-is).
  • Using 9relay or any shared backend — the app calls providers directly / on-device.
  • Accounts, telemetry, analytics.

8Constraints & non-negotiables

  • Privacy: OCR is on-device (Vision). Only recognized text is sent to a translate engine; images never leave the Mac. No telemetry.
  • Secrets: a user's OpenAI key lives only in the Keychain; never committed, logged, or written to the repo. No bundled author key.
  • No 9relay backend — it's a personal CLI relay, not a product backend. The app talks to providers directly (Google public endpoint / OpenAI with the user's own key).
  • Repo: lives in cdcupt/translator-everywhere under /macos; the te CLI + Quick Actions are untouched.
  • Build: this Mac builds macOS apps via DEVELOPER_DIR override + xcodegen (Xcode 26.5). Swift + SwiftUI/AppKit, no third-party services.
